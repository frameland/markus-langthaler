Supports Points, Lines, Triangles and Textured Quads.
Instead of the usual implementation with a single particle class that is inside an array, the data-oriented approach works the other way around. You have one array per particle property, such as vec2 position[MAX_PARTICLES] and float time[MAX_PARTICLES].



This has several advantages:
  • Easy submitting to the GPU, as the data is already in the right format
  • Data and behaviour are decoupled. This makes combining multiple behaviours together easy.
  • Generally better cache locality (CPU) because data is laid out continously in memory.

Each Particle System consists of 4 components:
  • Data: position, velocity, acceleration, color, time, alive, ...
  • Generators: PosRound, PosArea, Arc, LineFan, Time, DirectionForce, ...
  • Updaters: Time, Euler, FadeIn, FadeOut, ColorOverLifetime, Converge, FramesOverLifetime, ...
  • Renderer: Point, Line, Triangle, Quad


Generators run every time an emitter wants to emit particles.
E.g. The generator PosRound will set positions in a circular fashion.
Updaters will run each frame, e.g. Time will reduce the time property until it reaches 0 (and kills it).
The Renderer simply takes the particle Data and draws it to the screen.
